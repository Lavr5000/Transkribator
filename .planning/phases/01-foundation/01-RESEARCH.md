# Phase 1: Foundation - Research

**Researched:** 2026-02-05
**Domain:** Windows Installer (Inno Setup + PyInstaller)
**Confidence:** MEDIUM

## Summary

Phase 1: Foundation нацелен на создание базового Windows установщика для Transkribator. Исследование показало, что стандартным стеком для Python приложений с PyQt6 является связка **PyInstaller** (для упаковки в EXE) + **Inno Setup** (для создания установщика).

**Ключевые находки:**
1. PyInstaller поддерживает PyQt6 из коробки с версии 5.13+
2. Inno Setup автоматически создаёт деинсталлятор, регистрируя приложение в "Программы и компоненты"
3. Whisper модели (Sherpa/Whisper) должны загружаться при первом запуске, а не включаться в установщик
4. Для PyQt6 требуются special hooks при использовании PyInstaller

**Primary recommendation:** Использовать PyInstaller 6.0+ для компиляции Python приложения в standalone EXE, затем Inno Setup 6.3+ для создания установщика с базовыми страницами мастера.

## Standard Stack

### Core

| Library | Version | Purpose | Why Standard |
|---------|---------|---------|--------------|
| **PyInstaller** | 6.0+ | Компиляция Python в EXE | Стандарт де-факто для Python приложений, поддерживает PyQt6 |
| **Inno Setup** | 6.3+ (Unicode) | Создание Windows установщика | Бесплатный, ANSI + Unicode, работает на Win10/11 x64 |
| **PyQt6** | 6.5.0+ | GUI фреймворк (уже используется) | Требует special hooks в PyInstaller |

### Supporting

| Library | Version | Purpose | When to Use |
|---------|---------|---------|-------------|
| **PyInstaller hooks** | via pyinstaller | PyQt6 поддержка | Автоматически обрабатывает Qt plugins и зависимости |

### Alternatives Considered

| Instead of | Could Use | Tradeoff |
|------------|-----------|----------|
| PyInstaller | cx_Freeze | Менее зрелый для PyQt6, хуже поддержка |
| PyInstaller | Nuitka | Дольше компиляция, но потенциально быстрее EXE |
| Inno Setup | NSIS | Более сложный синтаксис, меньше примеров |
| Inno Setup | WiX Toolset | Только для MSI, избыточный для простых случаев |
| Inno Setup | MSIX | Требуется сертификат для подписи, сложнее настройка |

**Installation:**
```bash
# PyInstaller (для компиляции Python)
pip install pyinstaller>=6.0

# Inno Setup (скачать и установить)
# https://jrsoftware.org/isdl.php
# После установки: ISCC.exe доступен в PATH
```

## Architecture Patterns

### Recommended Project Structure

```
Transkribator/
├── build/              # Generated by PyInstaller (onedir mode)
├── dist/               # Generated by PyInstaller (onefile mode)
├── installer/          # Inno Setup scripts
│   ├── transkribator.iss     # Main installer script
│   └── assets/               # Installer icons, images
├── scripts/            # Build automation
│   ├── build_exe.bat         # PyInstaller command
│   └── build_installer.bat   # Compile .iss to .exe
├── main.py             # Entry point
├── src/                # Application source (уже существует)
└── pyproject.toml      # Dependencies (уже существует)
```

### Pattern 1: PyInstaller OneDir + Inno Setup

**What:** Сначала PyInstaller создаёт папку с EXE и зависимостями (onedir), затем Inno Setup упаковывает эту папку в установщик.

**When to use:** Для PyQt6 приложений — onefile создаёт огромный EXE (~200MB+) и медленный запуск из-за распаковки.

**Example .spec file:**
```python
# transkribator.spec - Generated then modified
# Source: PyInstaller documentation

a = Analysis(
    ['main.py'],
    pathex=[],
    binaries=[],
    datas=[
        ('src', 'src'),                      # Include source modules
        ('Transkribator_Pro.ico', '.'),      # Application icon
    ],
    hiddenimports=[
        'PyQt6.QtCore',
        'PyQt6.QtGui',
        'PyQt6.QtWidgets',
        'src.audio_recorder',
        'src.backends.sherpa_backend',
        'src.backends.whisper_backend',
        'src.transcriber',
        'src.hotkeys',
    ],
    hookspath=[],
    hooksconfig={},
    runtime_hooks=[],
    excludes=[],
    noarchive=False,
)

pyz = PYZ(a.pure)

exe = EXE(
    pyz,
    a.scripts,
    a.binaries,
    a.datas,
    [],
    name='Transkribator',
    debug=False,
    bootloader_ignore_signals=False,
    strip=False,
    upx=True,
    upx_exclude=[],
    runtime_tmpdir=None,
    console=False,          # No console window for GUI app
    disable_windowed_traceback=False,
    argv_emulation=False,
    target_arch=None,
    codesign_identity=None,
    entitlements_file=None,
    icon='Transkribator_Pro.ico',
)
```

**Build command:**
```bash
pyinstaller --clean transkribator.spec
```

### Pattern 2: Inno Setup Basic Script

**What:** Базовый скрипт Inno Setup для создания установщика с мастером из 4 страниц.

**Example .iss file:**
```pascal
; transkribator.iss
; Source: Inno Setup documentation + Jordan Russell's examples

[Setup]
AppName=Transkribator
AppVersion=1.0.0
AppPublisher=Lavr5000
AppPublisherURL=https://github.com/Lavr5000/Transkribator
AppSupportURL=https://github.com/Lavr5000/Transkribator/issues
AppUpdatesURL=https://github.com/Lavr5000/Transkribator
DefaultDirName={autopf}\Transkribator
DefaultGroupName=Transkribator
AllowNoIcons=yes
OutputDir=installer\output
OutputBaseFilename=Transkribator-Setup
Compression=lzma2
SolidCompression=yes
WizardStyle=modern
; ArchitecturesAllowed=x64
; ArchitecturesInstallIn64BitMode=x64

[Languages]
Name: "russian"; MessagesFile: "compiler:Languages\Russian.isl"
Name: "english"; MessagesFile: "compiler:Languages\English.isl"

[Tasks]
Name: "desktopicon"; Description: "{cm:CreateDesktopIcon}"; GroupDescription: "{cm:AdditionalIcons}"

[Files]
; Main executable and dependencies from PyInstaller
Source: "dist\Transkribator\*"; DestDir: "{app}"; Flags: ignoreversion recursesubdirs

[Icons]
Name: "{group}\Transkribator"; Filename: "{app}\Transkribator.exe"
Name: "{group}\{cm:UninstallProgram,Transkribator}"; Filename: "{uninstallexe}"
Name: "{autodesktop}\Transkribator"; Filename: "{app}\Transkribator.exe"; Tasks: desktopicon

[Run]
Filename: "{app}\Transkribator.exe"; Description: "{cm:LaunchProgram,Transkribator}"; Flags: nowait postinstall skipifsilent
```

**Build command:**
```bash
ISCC.exe installer\transkribator.iss
```

### Anti-Patterns to Avoid

- **OneFile для PyQt6 apps:** Создаёт огромный EXE (~200MB+), медленный запуск из-за распаковки во временную папку. Используйте OneDir.
- **Включение моделей в установщик:** Whisper/Sherpa модели весят сотни MB. Загружайте их при первом запуске через huggingface_hub.
- **Использование outdated PyInstaller (<5.13):** Не поддерживает PyQt6 корректно.
- **ANSI версия Inno Setup:** Не поддерживает Unicode в путях и русском языке. Используйте Unicode версию (isl файлы).

## Don't Hand-Roll

Problems that look simple but have existing solutions:

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Упаковка Python в EXE | Скрипт с zip/copy | PyInstaller | Обрабатывает import hooks, Qt plugins, hidden imports |
| Создание деинсталлятора | BAT/PowerShell скрипты | Inno Setup [UninstallDelete] | Автоматически регистрирует в Windows Registry |
| Регистрация в "Программы и компоненты" | Ручная запись в реестр | Inno Setup [Setup] | Создаёт правильные ключи HKLM\Software\Microsoft\Windows\CurrentVersion\Uninstall |
| Создание ярлыков | *.lnk через COM | Inno Setup [Icons] | Кроссплатформенно, обрабатывает спец. папки |
| LZMA сжатие | 7zip SDK | Inno Setup Compression=lzma2 | Встроено, оптимизировано |

**Key insight:** Inno Setup автоматически решает 90% задач Windows установщика — деинсталляция, реестр, ярлыки, сжатие, страницы мастера. Писать это вручную — ошибка новичка.

## Common Pitfalls

### Pitfall 1: PyQt6 Qt Plugins Not Found

**What goes wrong:** При запуске скомпилированного EXE падает с ошибкой "Qt plugins not found" или "Could not load Qt platform plugin 'windows'".

**Why it happens:** PyInstaller по умолчанию не включает Qt plugins (platforms/, imageformats/, etc.).

**How to avoid:**
```python
# В .spec файле:
import sys
from PyInstaller.utils.hooks import qt

# Автоматически включать Qt plugins
binaries = qt.binaries.get('PyQt6', [])
datas = qt.datas.get('PyQt6', [])

a = Analysis(
    # ...
    binaries=binaries,
    datas=datas,
)
```

**Warning signs:** EXE запускается и сразу закрывается, в логах "QWindowsIntegrationPlugin" или "platforms".

### Pitfall 2: Missing Hidden Imports

**What goes wrong:** ModuleNotFoundError при запуске EXE для модулей, которые импортируются динамически.

**Why it happens:** PyInstaller не может обнаружить динамические импорты (типа `importlib.import_module()`).

**How to avoid:**
```python
# В .spec файле добавьте hiddenimports:
hiddenimports=[
    'src.backends.sherpa_backend',
    'src.backends.whisper_backend',
    'src.backends.podlodka_backend',
    'src.audio_recorder',
    'src.hotkeys',
    'src.transcriber',
    'sherpa_onnx',
    'faster_whisper',
],
```

**Warning signs:** При запуске EXE: "ModuleNotFoundError: No module named '...'"

### Pitfall 3: Models Path Issues After Installation

**What goes wrong:** После установки приложение не находит модели (путь относительный, рабочая директория другая).

**Why it happens:** PyInstaller меняет `sys._MEIPASS`, а Inno Setup устанавливает в Program Files.

**How to avoid:**
```python
# В src/config.py или main.py:
def get_app_dir() -> Path:
    """Get application data directory for models."""
    if getattr(sys, 'frozen', False):
        # Running as compiled executable
        if hasattr(sys, '_MEIPASS'):
            # PyInstaller temp dir
            base_dir = Path(sys._MEIPASS)
        else:
            # Installed executable
            base_dir = Path(sys.executable).parent
    else:
        # Running from source
        base_dir = Path(__file__).parent.parent

    # Models should be in user data, not Program Files
    model_dir = Path(platformdirs.user_data_dir("Transkribator", "Lavr5000")) / "models"
    return model_dir
```

**Warning signs:** "Model files not found" при первом запуске установленного приложения.

### Pitfall 4: Antivirus False Positives

**What goes wrong:** Windows Defender или другой антивирус блокирует установку или запуск EXE.

**Why it happens:** PyInstaller bootloader часто детектируется как suspect.

**How to avoid:**
- Использовать UPX=False в .spec файле (UPX сжатие триггерит антивирусы)
- Подписать EXE с помощью code signing сертификата (если есть)
- Добавить исключение в Windows Defender для тестирования

**Warning signs:** При запуске установщика "Windows protected your PC" или исчезновение EXE после сборки.

### Pitfall 5: Russian Language in Installer

**What goes wrong:** Русский текст отображается как "?????" или квадратики.

**Why it happens:** Использование ANSI версии Inno Setup или отсутствие Russian.isl.

**How to avoid:**
```pascal
[Languages]
Name: "russian"; MessagesFile: "compiler:Languages\Russian.isl"
```

Убедитесь, что .iss файл сохранён в UTF-8 with BOM или UTF-8.

## Code Examples

Verified patterns from official sources:

### PyInstaller Build Script

```batch
REM build_exe.bat
REM Source: PyInstaller official documentation

@echo off
echo Building Transkribator EXE...

REM Clean previous builds
if exist build rmdir /s /q build
if exist dist rmdir /s /q dist

REM Build with PyInstaller
pyinstaller --clean ^
    --name=Transkribator ^
    --icon=Transkribator_Pro.ico ^
    --windowed ^
    --noupx ^
    --add-data="src;src" ^
    --hidden-import=PyQt6.QtCore ^
    --hidden-import=PyQt6.QtGui ^
    --hidden-import=PyQt6.QtWidgets ^
    main.py

if %ERRORLEVEL% NEQ 0 (
    echo Build failed!
    exit /b 1
)

echo Build complete: dist\Transkribator\
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| PyInstaller < 5.13 | PyInstaller 6.0+ | 2023 | Полная поддержка PyQt6 |
| ANSI Inno Setup | Unicode Inno Setup | 2021 | Поддержка русского языка |
| Manual registry writing | Inno Setup auto-registration | Стандарт | Надёжность |
| OneFile bundles | OneDir + installer | Всегда | Быстрее запуск, меньше проблем |

**Deprecated/outdated:**
- **PyInstaller 3.x:** Не поддерживает Python 3.12+ и PyQt6
- **NSIS:** Сложнее синтаксис, меньше примеров для PyQt6
- **MSIX без сертификата:** Требуется подпись для Windows 10/11

## Open Questions

1. **Sherpa-ONNX runtime dependencies**
   - What we know: Sherpa-ONNX требует onnxruntime.dll
   - What's unclear: Нужно ли включать onnxruntime.dll в установщик или она подтягивается автоматически
   - Recommendation: Проверить PyInstaller анализ зависимостей sherpa_onnx, добавить `--hidden-import=onnxruntime` при необходимости

2. **huggingface_hub cache location**
   - What we know: Модели загружаются через huggingface_hub.snapshot_download()
   - What's unclear: Где будет кеш после установки в Program Files (Permissions?)
   - Recommendation: Использовать platformdirs.user_data_dir() для моделей, избегать Program Files

3. **UPX compression compatibility**
   - What we know: UPX уменьшает размер EXE, но может триггерить антивирусы
   - What's unclear: Стоит ли использовать UPX для финальной сборки
   - Recommendation: Для Phase 1 использовать `--noupx` для простоты, UPX оставить для оптимизации
